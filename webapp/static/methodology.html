Methods Overview

Data & Features

Daily stacks are exported from Google Earth Engine as multi-band GeoTIFFs with the following bands (in order):

  • TempC, U10, V10, WindSpeed
  • NDVI
  • BurnDate (used to derive current burn mask)
  • LULC, DEM, Slope, Aspect, Hillshade

Models

  • Random Forest (ANY_BURN) on tabularized pixel samples with lagged dynamic features and static context.
  • U-Net Scenario 1 (operational model): predicts next-day burn probability from a single-day stack using 128×128 patches.
  • Additional experiments with multi-day U-Net scenarios and ConvLSTM sequences.

Training & Evaluation

  • Class imbalance handled via negative downsampling.
  • Decision thresholds chosen by maximizing G-mean / F1 on a held-out test set.
  • Metrics include ROC AUC, PR AUC, sensitivity, specificity, precision, and CBA.

For full experimental details, hyperparameters, and additional figures, see the Model.ipynb notebook in the repository.
